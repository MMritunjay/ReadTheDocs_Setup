    Entering Payment Voucher

    1. From the Transactions main menu select Voucher Entry.

    2. Select the Voucher Type as Payment from the drop down.

    Now a new panel appears.

    3. The voucher page displayed is in 3 partitions. The top one shows fields Voucher Number, Date (Denotes Voucher Date), Instrument Type, Instrument No, Instrument Date, Cash/Bank (From the account you are making payment), Address (To which account you are making payment), Amount, Remarks and a ‘Balance’ on either side below Cash/Bank and Address columns. The second partition shows Business Area, Profit Centre/Cost Centre, Debit/ Credit, Amount, Narration and an Journalise button. The third partition shows fields Dr., Cr., Diff. and buttons Search, Clear and Close.

    4. Fill in the following details on the first partition:

    • Date

    • Select Cash/Bank Account from which the payment is done.

    • Select Address from the dropdown against whom payment is to be made

    o Current balance before saving the entry will be shown in the ‘Balance’ field. Suffix D or C in brackets to the balance figure shown indicates whether the balance is Debit or Credit

    • Instrument Type

    • Instrument No

    • Instrument Date

    • Amount

    • Remarks

    5. Click Journalise button.

    Now, a credit entry appears in the second partition, an auto-generated narration appears in the narration column, the account selected against the field ‘Address’ appears along with ‘D’ against ‘Dr/Cr’ column, amount and auto-narration. The auto-narration field is editable so that user required narration can be entered. An ‘Add’ button replaces the ‘Journalise’ button. One more button Save appears in the second partition. The Dr. or Cr. and Diff. fields get updated.

    6. Select the Business Area, Profit Centre, and Cost Centre from the respective dropdowns in the second partition.

    7. Click Add

    Now, two new fields Delete and Cancel appear on the second partition. A debit entry is generated on the third partition

    8. Click Save button to add the voucher entry.

    System checks whether the Debit and Credit are equal; if not throws a message ‘Debit and Credit are not equal’, and the voucher is not saved. if yes, the voucher is saved. The added voucher entry is displayed in the third partition.

    9. A voucher number is generated and shown against the ‘Voucher Number’ field

    10. A pop up message ‘Do you want to print this report’.

    11. Click ‘Cancel’ if you do not want to print it immediately.

    12. Click Ok to print, to get the printer interface.

    13. Click Ok, the voucher gets printed.

    14. Along with the pop up message, a pop up page named Set Off Wizard (Voucher Audit) page appears. This page is used to map debit entries with credit entries.*

    15. Buttons Print Report, View Report, Print Cheque can be seen along side buttons Search, Clear and Close.

    16. Print Report will direct you to the printer interface and view report will give you a print preview of the voucher.

    17. Click Clear, and user will be able to do a fresh contra voucher on the same voucher date. Click Close, the contra voucher format is closed and user can now select a new voucher type from the voucher type combo.

    18. Click Print Cheque, a pop up PRINT CHEQUE appears where vendor name is defaulted to the ‘Address’ selected during voucher entry. It can be modified also.

    19. Click Print to print the cheque **.

    * Detailed functionality of this is narrated in Point No.4.6

    **Printer configuration and system configuration are narrated separately in point no.
