    3.2 Per Visit Activities - OP

    3.2.1 Creating Visit

     use 'Create Visit Screen' to create a visit.

     Type MRD No. And Tab Out.

     Select 'Visit Type'

     Use 'Save'

    3.2.2 Mapping Insurnace to a Visit

     Use 'Visit Insurance Map' button.

     A pop up 'Admission Policy Map' will come, where you can see if any scheme is already mapped. If no scheme is mapped use 'Add New Insurance' button.

    Here ECHS Polyclinics are treated as an insurance company. If no ECHS polyclinic is not mapped to the patient previously, you need to add the patient’s polyclinic details using the ‘Add New Button. If any policlinic is added earlier, you can see it using the Visit Insurance Map browser. Fill in the required details as shown in the screen shot below.

    Save the details

    If any details need to be edited, you can use the Edit button. You can inactivate the mapping by removing the check box against Active field. Once added you can see the added policy (Polyclinic) in the Visit Insurance Main page it self. Select the policy and use Assign to Visit, as shown in the screen shot below, to attach the policy to the visit. Note only Primary policies can be assigned to a visit. That is if a patient is having multiple insurance policies mapped to his name, the user has to select one policy as the primary policy and map it against the visit

    3.2.3 Mapping Patient Category and Billing Type to the Visit

    Use 'Change Visit Details' in the 'Create Visit' page to Map 'Patient Category' and 'Billing Type' to the visit. Now since the billing type is set the rates applicable to the mapped billing type will be taken by the system from the service master. All the rates from Encounter to other services will be taken from the rates asigned to the billing type in the service master.

    3.2.4 Setting Credit Limit

    Use 'Set Credit Limit' button in the 'Create Visit' page to set credit limit for the patient. The credit limit can be 'No Limit', 'Limited Credit' and 'Unlimited'. The credit limit can be separately set for Services and Pharmacy/Material. The credit limit will is shown in all billing pages of the application so that the users are aware of the same and act accordingly.

    Note:- Total credit Limit for the patient will be Cash Paid by the patient, Insurance Approved amount as set in the authorization and the credit limit explicitly set using the set credit limit button

    3.2.5 Encountering the Patient

    Before the patient consults the doctor, an encounter will be done in the system.

    3.2.6 Capturing Authorization Details

    Once the encounter is created, the authorization received from the insurance company/ECHS need to be saved in the system. Please follow the steps below:-

     Select the Authorization Request page

    If any insurance (Polyclinic) is mapped to the visit Corporate Name and Policy will get displayed in the page automatically. Fill in the details as in the Authorization form. Please see the screen shot below

    Note:- Please make sure that in the Diagnosis and Treatment Plan, type the contents as continuous text. Please do not use Enter Keys since the text so entered will create alignment issues when printing the Appex C.

    Use Authorization Request Browser to view and edit already saved Authorizations.

    Use Edit Button to open the saved authorization details.

    Use Approve to complete the task.

    Note:- Make sure that the Authotization is set to Approve since Authorization number and Date of the approved authorizations will be shown in the required column of Appex C

    Adding Authorization details is a one time activity for a particular visit.

    3.2.7 Billing

    Use OP Bill menu, auto complete description fields to do the billing. Once saved, the services will be shown under the correct category to which it is mapped using the Revenue Code Master and Item Revenue Code Map
