    Reorder Management:

    For maintaining proper supply, any inventory system has to monitor and maintain the stock levels on check. Inventory level has to take care of sudden increase in demand as well as decrease in demand. Many factors contribute to determine the level of stock at which next order has to be placed and the quantity to be ordered like consumption patterns, lead time etc.

    Reorder functionality in Amrita HIS

    The application supports setting reorder values for each individual item by location (store). This means the item can have a different reorder point for each store.

    User could either set the values through master screen or could configure to automatically update the values in reorder.

    Reorder Master Screen

    The master screens for reorder is present under the menu - Pharmacy/ Material Item Reorder (under MM Masters menu).

    The screen has 3 tabs -

    StockClass Master,

    Item - Store - StockClass Map and

    Reorder Master.

    StockClassMaster Tab

    A stock class defines a template of reorder properties and can be mapped to any item if it has those values. The key fields for stock class are:

    Daily Consumption Algorithm

    Minimum Days

    Maximum Days

    Avg. Lead Time

    Days to Sum (Used only for Algorithm 8)

    Algorithms available for calculating avg. daily consumption:

    Average daily consumption for the past year(for an item displaying very stable consumption pattern)

    It is calculated by taking the sum of total issue qty, total transfer qty and total consumed qty for the last one year.

    Calculation: Avg. Consumption = total / 365

    Average daily consumption for the past month(for item which has changing but unpredictable consumption pattern)

    It is calculated by taking the sum of total issue qty, total transfer qty and total consumed qty for the last month.

    Calculation: Avg. Consumption = total /no of days of last month

    Average daily consumption for the same month previous year (for item which has changing but seasonal consumption pattern

    It is calculated by taking the sum of total issue qty, total transfer qty and total consumed qty for the same month last year.

    Calculation: Avg. Consumption = total /no of days of same month last year.

    Maximum of the Average daily consumption for the last 12 monthly buckets (for old items which has to be continued and has not much seasonal pattern)

    It is calculated by taking maximum of the sum of total issue qty, total transfer qty and total consumed qty for the last 12 months.

    Calculation: Avg. Consumption = maximum of avgConsumption during last 12 months.

    Minimum of the Average daily consumption for the last 12 monthly buckets (for old items which needs to be phased out)

    It is calculated by taking minimum of the sum of total issue qty, total transfer qty and total consumed qty for the last 12 months.

    Calculation: Avg. Consumption = minimum of avgConsumption during last 12 months.

    Maximum daily consumption of the last four 15 day buckets (for costly items having erratic consumption pattern)

    It is calculated by taking maximum of the sum of total issue qty, total transfer qty and total consumed qty for the last 15 days.

    Calculation: Avg. Consumption = maximum of avgConsumption during last four 15 days.

    Last 2 months average (for items having fairly constant/stable consumption pattern)

    It is calculated by taking average of the sum of total issue qty, total transfer qty and total consumed qty for the last 2 months.

    Calculation: Avg. Consumption = total /no of days of last two months.

    Average daily consumption for the given number of days

    It is calculated by taking average of the sum of total issue qty, total transfer qty and total consumed qty for given no of days.

    Calculation: Avg. Consumption = total /no of days ( Days To Sum entered in Item Stock Class Master)

    Table used: item_stock_class_master

    Item - Store - StockClass MapTab

    System supports reordering at each store level, if required. For this, map the item, store and stock class. The exec frequency has Manual, Daily, Weekly, Monthly, in 6 month, yearly. The exec.frequency other than Manual is used for cron job. The cron job will look at the cron frequency and calculate and insert the reorder values.

    Table used: item_store_stockclass_map

    Reorder Master Tab

    Through the Reorder Master tab, user could manually set the reorder values of an item store wise. The log in store is defaulted in the Reorder Master screen.

    Reorder Master Screen has fields to capture Daily Consumption, Minimum Quantity, Maximum Quantity, Reorder Level(ROL) and Reorder Quantity (ROQ).

    If the item has Item Store StockClass Map entry and the exec.frequency is set as Manual, when the item is selected, the reorder values are calculated automatically and shown in the fields Daily Consumption, Minimum Qty, Max.Qty Reorder Level and Reorder Qty. Then the values can be saved.

    There is an icon present near the Daily Consumption field. The button is for the user to check and get the latest avg. daily consumption value if present any and to calculate the latest reorder value based on it. User could then modify the details. For this to work, need to do the stock class and store stock class mapping as

    well as the item should have any consumptions happened in the store.

    Table used: item_reorder.

    Screens showing Reorder details

    The reorder details captured at the master level is shown in stock browsers and used in indent.

    Stock Browsers

    The ROL, ROQ, Min. Qty and Max. Qty are shown in the Main Stock Browser as well as in the mini stock browsers present in the indent and PRQ screens.

    The stock browsers show the local stock colored, if the stock is below ROL.

    Purchase Request

    User could search by items that are 'Below ROL' in the stock browser present in the PRQ screen. And purchase request could be created by adding the Below ROL items.

    Indent

    In the Indent screen, on adding the items that are 'Below ROL' from stock browser, the ordering qty is defaulted as (Max.Qty - Stock In Hand).

    Reorder Calculations

    Reorder level:

    ROL = Lead Time Demand + SafetyStock

    where,

    Lead Time Demand = Lead Time * Avg. Daily Consumption

    Avg. Daily Consumption = Total qty moved out of the store / total days. (Default).

    Avg. Daily Consumption = (Max. Qty + Min. Qty) / 2 (If value of app.config - MM-REORDER_AVGCONSUMPTION_METHOD is MINMAXAVG).

    Safety Stock has multiple calculations supported:

    Safety Stock = Min.Qty (Default)

    Safety Stock = Max of Consumption for the period * Lead Time (If value of app.config - MM-REORDER_SAFETYSTOCK_METHODis LEADTIMEMAX)

    Reorder Quantity:

    By default,

    ROQ = Max. Usable Qty - Min. Usable Qty

    where,

    Max. Usable Qty = Maximum Days * Avg. Daily Consumption

    Min. Usable Qty = Minimum Days * Avg. Daily Consumption

    ROQ = Max. Usable Qty - Local Stock ( if value of MM-REORDER_SAFETYSTOCK_METHOD is LEADTIMEMAX )

    ROQ value set in reorder master is not used in the application.

    Reorder Level Reports

    Minimum Level Report for Materials with PRQ-PO Details

    Report will show the PRQ and PO details of items that are below ROL. The report will list data only if any PRQ or PO is present for the item that are below reorder level.

    Terminology

    Safety Stock: This is the minimum stock balance that you want to have on hand at all times. The actual value will be heavily based on how critical the inventory item is to your operation.

    Reorder Point\ Level (ROP): Reorder point is the balance level at which you want your inventory reorder to take place for the given item. The calculation is a forecast of demand over lead time plus a calculation of safety stock.
