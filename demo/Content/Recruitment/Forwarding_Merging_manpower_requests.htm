    Forwarding Merging manpower requests

    All approved requests are to be forwarded to the Export List, from where the request can be exported to the website application.

    There are 2 scenarios concerning the approved requests.

     There may be just a single request raised by a department for vacancies in a particular position

    OR

     It is possible that individual requests will be raised by multiple departments re-garding vacancy for the same position.

    For e.g. multiple clinical departments may have need for more nurses with expe-rience.

    Since requirements, evaluation criteria and process will be same, there need be just a single request and a single corresponding notification inviting applications.

    In Amrita HIS, the privileged user can merge selected requests and ensure that there are no duplicates requesting for the same vacancies.

    This merging happens during the forwarding of requests:

    1 From HR Recruitment menu select Manpower Request sub menu.

    Here, select Forward Requests for export

    2 Forward Manpower Requests for export page opens

    3 Enter Designation and date range (of creation of request)

    4 Click Search

    5 Single/multiple approved requests satisfying the given criteria are listed.

    6 Please note that requests once already exported will not be displayed here

    7 Click on the button “Forward”

    8 In case there is just a single request then it is forwarded directly to Export list.

    Merging manpower requests

    1 In case there are multiple requests, then a new child window Merge Manpower Requests opens for user to fill in the new request details. Here, the no: of vacancies displayed is the sum of approved vacancies in the multiple merging requests

    2 Enter the other details of the merge – Eligibility, Required before (date), salary, experience etc.

    3 Click Save

    4 Manpower Requests – Export List page opens

    5 The list here displays details of requests already exported as well as those yet to be exported as indicated in the status column for each line item.

    6 The forwarded single/merged requests appear here as different line items
